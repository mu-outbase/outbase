# OUTBASE Record-01
## 記録画面 項目・操作・詳細遷移 承認案 v1

作成日：2026-07-11  
状態：実装前承認案  
UIベース：OUTBASE_ROUTE08_UI_REBUILD_CLEAN_v6_基準寄せ承認版  

---

## 1. 今回の目的

CLEAN v6で正式承認されたデザイン骨格を変更せず、記録画面に必要な項目・文言・操作・保存動作・詳細遷移を確定する。

記録画面は管理画面ではなく、現地で3秒以内に記録を残す道具とする。

---

## 2. 画面の主役

記録画面の主役は次の3つだけ。

1. 今どこへ保存しているか
2. 現在の地図・GPS・セッション状態
3. 話す / 撮る / 動画 / 場所 / ピン / メモ

分類・整理・タグ付け・AI確認は現地画面の主役にしない。

---

## 3. 上部表示

### 3.1 記録先表示

通常時：

```text
記録先　コタ通常散歩
```

親子がある時：

```text
記録先　赤城山キャンプ ＞ コタ場内散歩
```

保存先不明時：

```text
記録先　未確認箱へ仮保存
```

### 3.2 固定ルール

- 記録先は常に見えるが、大型カードにはしない。
- 表示中プランを切り替えても記録先は変えない。
- 記録先変更は、記録先表示をタップした時だけ行う。
- 候補シートは開きっぱなしにしない。
- 保存のたびに確認ポップアップを出さない。

### 3.3 補助状態

小さく次を表示する。

```text
GPS取得中 / オフライン保存 / 画面ON
```

異常時のみ表示を変える。

```text
GPS弱い / GPS欠測あり / 同期待ち / 容量注意
```

## 4. 地図エリア

### 4.1 表示内容

- 実地図
- 現在地
- 移動軌跡
- 写真地点
- 動画地点
- 音声文字起こし地点
- メモ地点
- 場所保存地点
- ピン地点
- GPS欠測区間

### 4.2 MAP切替

`MAP切替`で次を切り替える。

1. Google Map相当の実地図
2. サイトMAP（取得済みの場合）
3. 地図＋記録地点

サイトMAPがない場合は、存在しない切替候補を出さない。

### 4.3 地図操作

- 現在地：現在位置へ戻る
- ピン：現在地へ汎用ピンを即保存
- Google Map：外部地図を開く
- 地図上の記録点タップ：記録詳細の小型シートを開く

---

## 5. GPS・セッション値

### 5.1 基本4項目

散歩・移動系では次を表示する。

```text
GPS点 / 距離 / 経過時間 / 記録数
```

現在の仮項目「ピン数」は「記録数」へ変更する。

理由：記録画面全体の成果が分かり、写真・音声・メモも含められるため。

### 5.2 活動別表示

- 散歩：GPS点 / 距離 / 経過時間 / 記録数
- 移動：GPS点 / 移動距離 / 経過時間 / 寄り道数
- 設営・撤収：工程数 / 実働時間 / 休憩時間 / 記録数
- 料理：工程数 / 経過時間 / タイマー / 記録数
- セッションなし：位置精度 / 現在時刻 / 同期状態 / 記録数

同じ画面骨格を使い、活動タイプに応じて中身だけ変える。

## 6. セッション操作

### 6.1 未開始状態

主ボタン：

```text
記録開始
```

候補セッションが明確な場合：

```text
散歩開始
設営開始
料理開始
移動開始
```

候補が不明な場合は、汎用の「記録開始」にする。開始前でもクイック記録は使用可能。

### 6.2 記録中

主ボタンは状態に応じて変える。

```text
一時停止
再開
```

終了は誤操作防止のため、主ボタンと分けた小型操作にする。

### 6.3 終了

終了操作後は、分類画面へ強制遷移しない。

終了シートに表示するもの：

- セッション名
- 開始・終了時刻
- 経過時間
- 距離または工程
- 写真 / 動画 / 音声文字起こし / メモ / ピン件数
- GPS欠測
- 同期状態

操作：

```text
終了して保存
記録へ戻る
```

保存後の詳細整理は任意とする。

## 7. クイック記録6項目

順番はCLEAN v6の承認画面を維持する。

```text
話す / 撮る / 動画 / 場所 / ピン / メモ
```

### 7.1 話す

表示文言：

```text
話す
文字起こしメモ
```

動作：

- 長押し中だけ音声入力
- 指を離すと文字起こしメモとして即保存
- 分類選択は出さない
- オフライン時は端末内に一時保持し、通信復帰後に文字起こし
- ユーザーの主記録は文字起こしテキスト
- 誤変換は後から修正可能

### 7.2 撮る

表示文言：

```text
撮る
写真を残す
```

動作：

- 端末の標準カメラを起動
- 撮影後にOUTBASEへ戻る
- 撮影時刻・GPS・記録先へ自動紐付け
- 撮影キャンセル時は空記録を作らない
- 連続撮影用のアプリ内カメラは後続詳細設定に残す

### 7.3 動画

表示文言：

```text
動画
動画を残す
```

動作：

- 端末の標準動画撮影を起動
- 戻った時に時刻・GPS・記録先へ自動紐付け
- 動画内音声は動画素材の一部
- 音声文字起こしメモとは分離する

### 7.4 場所

表示文言：

```text
場所
今いる場所を保存
```

動作：

- 現在位置を場所候補として保存
- 名前入力は必須にしない
- GPS・時刻・周辺候補を保存
- 後から駐車場、トイレ、日陰、水場、店、危険箇所などへ昇格可能

### 7.5 ピン

表示文言：

```text
ピン
現在地に印を付ける
```

動作：

- 1タップで現在地へ汎用ピンを即保存
- 種類選択は必須にしない
- 保存後に任意の小型候補を表示可能
- 候補を選ばなくても記録は完了する

### 7.6 メモ

表示文言：

```text
メモ
文字で残す
```

動作：

- 小型入力シートを開く
- 入力中もGPSセッションは止めない
- 保存先を再選択させない
- 空欄では保存しない

## 8. 保存動作

### 8.1 保存先優先順位

1. ユーザーが明示したrecordTarget
2. 現在進行中のprimaryRecordingSession
3. GPS・時刻・予定が一致する高信頼候補
4. 未確認箱

### 8.2 自動紐付け

高信頼時：

```text
コタ通常散歩に保存しました
```

候補が複数ある時：

```text
仮保存しました
候補：赤城山キャンプ / コタ散歩
```

ただし記録を止めない。候補選択は後回しにできる。

### 8.3 保存時に持つ情報

共通：

- recordId
- rootPlanId
- sessionId
- recordTargetId
- type
- createdAt
- lat / lng
- GPS精度
- sourceDevice
- syncStatus
- confidence
- privacy

素材がある場合：

- localUri
- remoteUri
- provider
- hash

## 9. GPS欠測・オフライン・復元

### 9.1 GPS欠測

- GPSが弱くても記録を止めない
- 欠測開始・復帰時刻を保存
- 地図上に欠測区間を表示
- 推定補間した場合は推定値として区別
- ユーザー保存値を勝手に確定値へ置換しない

### 9.2 オフライン

- 写真・動画・文字起こし前音声・メモ・GPS・ピンを端末保存
- 通信復帰後に差分同期
- 同期失敗で記録を消さない
- 自動削除しない
- 容量不足時も撮影禁止ではなく段階警告

### 9.3 アプリ中断・強制終了

- activeSessionを復元
- GPS記録再開候補を出す
- 二重セッションを作らない
- 最後の保存済みRecordから復元

---

## 10. 画面ON維持

- セッション開始中は画面ON維持を有効にする
- 状態を小さく表示する
- バッテリー低下時は警告するが、勝手にセッションを終了しない
- ユーザーが画面を消しても、許可されている範囲でGPS記録を継続する

---

## 11. 記録詳細シート

地図上の点、記録数、直近記録をタップした時に開く。

表示：

- 記録種別
- 時刻
- 場所
- 記録先
- 写真 / 動画 / 文字起こし / メモ本文
- GPS精度
- 同期状態
- 関連候補

操作：

- 記録先を変更
- 複数対象へ関連付け
- テキスト修正
- 非公開へ変更
- 削除候補へ移動

削除は即時確定せず、復元可能にする。

## 13. 禁止事項

- UIデザインを作り直さない
- 記録画面をカード一覧化しない
- 保存先を毎回聞かない
- プランを作らないと記録できない設計にしない
- GPS欠測で記録を止めない
- 音声記録を分類入力画面にしない
- 写真撮影後に毎回タグ入力させない
- 表示対象切替でrecordTargetを変更しない
- 未確認箱を通常保存先として乱用しない
- 04.8ルート、FIELD03、既存GPS・写真・動画・音声・ピン・復元を壊さない
- 小出し修正をしない

---

## 14. 実機合格条件

1. 記録開始まで3秒以内
2. 話す / 撮る / 動画 / 場所 / ピン / メモが一画面に見える
3. 地図・GPS値・操作・クイック記録が一画面に収まる
4. 表示対象を変えても記録先が変わらない
5. GPS欠測でも写真・メモ・ピンが保存できる
6. 予定なしでも未確認箱へ記録できる
7. アプリ再起動後に記録中セッションを復元できる
8. オフライン記録が消えない
9. 画面ON維持が動く
10. 04.8ルート機能が壊れていない
11. CLEAN v6の承認デザインが崩れていない
12. 実機スクショ上で変更が確認できる
