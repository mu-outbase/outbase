# AGENTS.md — OUTBASE Codex作業ルール v140

## 最優先ルール

OUTBASEは、機能一覧アプリではなく、キャンプ前・当日・帰宅後の体験を資産化するアプリである。

Codexは次を守ること。

1. MVPα1の1ファイル構造へ戻さない。
2. 1つの修正で複数ファイルに影響する場合は、関連モジュール単位で整理する。
3. 手入力中心の実装を提案しない。自動取得、候補表示、取込、承認を優先する。
4. 予約スクショ読取だけで完了扱いしない。次のキャンプ準備候補へ接続する。
5. 散歩、キャンプ準備、記録、レビュー、次回改善の接続を壊さない。
6. PWAを前提に、オフライン起動・自動保存・復元の設計を壊さない。
7. 変更後は `docs/TEST_CHECKLIST_v140.md` と Core 01 追加チェックの該当項目を確認する。
8. カレンダーは「予定作成の入口」として扱い、Googleカレンダー直連携は認証設計後に行う。

## 実装対象

- `src/modules/*`：画面・機能モジュール
- `src/core/*`：ルーティング、保存、状態管理
- `src/domain/schema.js`：OUTBASEデータ構造
- `styles/app.css`：共通UI
- `docs/*`：仕様・移行・テスト

## 禁止

- 巨大な `index.html` に戻す
- 手入力フォームだけで機能完了にする
- OCRやAI候補を確定データとして自動登録する
- テストや確認なしにPWAキャッシュを強める
- MVP完成と断言する
