OUTBASE v19.1 / R3-02 新実行画面白画面修正

基準: GitHub main v19.0
目的: 準備画面の「活動を始める」から新実行画面へ遷移した際の白画面を解消する。

原因
- 旧シェル描画器は record ルート専用の本文を持たない。
- v19.0 は record でも先に旧描画器 mount() を呼んだ。
- record のモデルには HOME データが無いのに、旧描画器が HOME 描画へフォールスルーし、実行画面を描く前に例外終了した。

修正
- record ルートでは旧描画器 mount() を呼ばない。
- シェルモデルと共通ヘッダー／下ナビを直接準備してから、新実行画面を描画する。
- 予期しない例外時も白画面にせず、準備へ戻れるエラー画面を表示する。
- version / manifest / router / CSS / service worker のキャッシュ識別子を v19.1 へ更新する。

非変更
- 旧FIELD03
- GPS履歴、写真、動画、音声、ピン、IndexedDB
- HOME、カレンダー、予定詳細、準備保存
- 下バー構成

GitHub mainへの書込みは行っていない。Android実機は未確認。
