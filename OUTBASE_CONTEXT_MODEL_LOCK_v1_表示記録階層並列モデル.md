## 0. このLOCKの目的

04.8でルートAPIなし版は一旦OKになった。  
その後、04.9〜04.12でプランチップ、保存先保護、入口整理、表示中/記録中分離を進めたが、ユーザー実運用の想定が浅く、後追い修正が増えた。

このLOCKでは、OUTBASEの根本モデルを固定する。

固定するもの:

- OUTBASEは単一プラン切替アプリではない
- OUTBASEはキャンプ、散歩、準備、記録、整理、改善が並列に動く外出OSである
- 表示対象と記録対象は別概念
- 親子孫までで内部構造を制限しない
- UIでは深さを抑え、内部では可変階層 + 複数リンクを許可する
- APIあり移行でも保存値を勝手に上書きしない
- 未確認箱は通常運用ではなく保険

## 2. OUTBASEの根本定義

OUTBASEは「現在プランを1つ選んで、その中だけ操作するアプリ」ではない。

OUTBASEは、次のような現実の外出運用を扱う。

```text
今のキャンプを実行しながら、次のキャンプを準備する
キャンプ準備中にコタ散歩をする
コタ散歩中にキャンプの買い物を確認する
キャンプ中に場内散歩を記録する
料理中に写真・音声・タイマー・ギアメモを残す
帰宅後レビュー中に次回候補探しをする
予定を作る前に写真や音声だけ先に残す
```

したがって、OUTBASEは以下を分ける。

```text
表示対象：今見ている予定・作業・情報
記録対象：今保存する先のセッション
親予定：キャンプ、散歩、ピクニック、ギア整備などの大枠
子セッション：設営、撤収、場内散歩、料理、移動、買い物などの行動単位
記録素材：写真、動画、音声、メモ、GPS、ピン、タイマー結果など
関連リンク：1つの記録が複数の予定・場所・改善点に関係する紐付け
未確認箱：判断不能時の保険
```

## 3. 最重要LOCK

### 3.1 表示対象と記録対象は分ける

```text
表示中：今見ているもの
記録中：今保存しているもの
```

例:

```text
表示中：次回キャンプ準備
記録中：赤城山キャンプ / コタ場内散歩
```

この状態で:

- 準備・ギア・買い物・料理・ルートは次回キャンプを見られる
- 写真・音声・GPS・ピンはコタ場内散歩に保存できる
- 表示を変えても保存先は勝手に変わらない

### 3.2 表示切替で保存先を変えない

禁止:

```text
散歩記録中に赤城山準備を見た
↓
保存先が勝手に赤城山へ変わる
```

正:

```text
散歩記録中に赤城山準備を見た
↓
表示中だけ赤城山
↓
記録中はコタ散歩のまま
```

### 3.3 保存先変更は明示操作だけ

保存先を変える操作には、最低限わかる表示が必要。

```text
記録先を変更
今後の写真・音声・GPSを「赤城山」に保存します
```

ただし毎回確認ポップアップは禁止。  
通常は軽い表示でよい。

### 3.4 未確認箱は保険

未確認箱は便利な通常保存先ではない。  
判断不能時、予定未作成時、GPS欠測時、保存先不明時にだけ使う。

禁止:

```text
面倒だから全部未確認箱へ
プラン選択しないと記録できない
保存先不明なので記録失敗
```

正:

```text
保存先が高信頼なら自動で紐付け
低信頼なら未確認箱へ仮保存
あとで整理・移動・複数リンク可能
```

## 4. 階層モデル

### 4.1 UIは最大3階層程度に抑える

画面で常時見せるのは、原則として以下まで。

```text
親：赤城山キャンプ
子：コタ場内散歩
孫：写真 / GPS / ピン / メモ
```

これは表示上の制限であり、データ上の制限ではない。

### 4.2 内部データは可変階層

内部は孫までに限定しない。

例:

```text
赤城山キャンプ
└ 1日目
   └ 設営
      └ レイアウト確認
         └ 写真
         └ 音声メモ
         └ 風対策メモ
```

ただし、UIで深い階層を常時見せない。  
普段は「いま見るもの」「いま記録するもの」だけ見せる。

### 4.3 ツリーだけではなくグラフ構造を持つ

1つの記録が複数対象に関係することがある。

例:

```text
写真A
→ 赤城山キャンプ
→ コタ場内散歩
→ 木陰スポット
→ 次回改善
```

したがって、OUTBASEの内部構造は:

```text
可変階層 + 複数関連リンク
```

とする。

## 5. エンティティ定義

### 5.1 Plan / Project

外出・予定・大枠。

例:

- スノーピーク赤城山CF 2026/7/18-20

```text

title
start / end
place
status: planned / preparing / active / paused / done / cancelled
parentId: 基本null
createdAt / updatedAt
privacyDefault
```

### 5.2 Session / Task

Planの中で起きる行動単位。

例:

```text

├ 設営
├ 1日目夜ごはん
```

```text
status: planned / active / paused / done
startedAt / endedAt
placeId
privacyDefault
```

### 5.3 Record

出来事や入力ログ。

例:

- 写真を撮った
- 音声メモを残した
- GPS点を保存した
- ピンを置いた
- タイマーが完了した
- メモを書いた

実ファイル・素材。

例:

- 写真ファイル
- 動画ファイル
- 音声ファイル
- GPX
- サムネイル

推奨フィールド:

```text

```

### 5.5 Link

複数関連を表す。

例:

```text

```

```text

relation: belongsTo / alsoRelated / evidenceOf / improvementFor / locationOf / usedIn / createdDuring

```

```text

parentPlaceId

```

### 5.7 Gear / Meal / Checklist

ギアや料理はPlan配下のTaskにもなり、Record/Linkの対象にもなる。


## 6. 画面コンテキスト定義

### 6.1 UI状態

```text
viewTargetId: 今見ている対象
viewTargetType: plan / session / place / gear / meal / review / inbox
recordTargetId: 今保存する先
recordTargetType: plan / session / inbox
activeSessions: 現在動いているセッション一覧
primaryRecordingSessionId: クイック記録の主保存先
referenceTargets: 一時参照している対象一覧
```

### 6.2 上バー表示

大きくしない。小型表示。

基本:

```text
表示中：赤城山準備
記録中：コタ場内散歩
```

条件:

- 表示中と記録中が同じなら1行でよい
- 違う時だけ2つを軽く表示
- デカい現在プランカードは禁止
- 横チップ常時表示は禁止
- 開きっぱなしの切替シートは禁止

### 6.3 下バー

下バーは固定。

```text
予定 / 探す / 準備 / 記録 / 思い出
```

プランによって下バーの意味を変えすぎない。  
ボタン名を頻繁に変えない。

- 準備: viewTargetの準備を見る
- 記録: recordTargetへ記録する
- 予定: 全体予定・表示対象切替
- 探す: 候補・ナレッジ
- 思い出: 記録・整理・レビュー

### 6.4 クイック記録

記録画面は以下に集約。

```text
話す
撮る
動画
場所
ピン
メモ
```

保存先はrecordTarget。  
表示対象を見ているだけではrecordTargetを変更しない。

### UC-06 移動中に寄り道・買い物・到着記録


```text

```

合格条件:

- 記録不能にしない
- 後でPlan/Sessionへ移動できる

### UC-08 帰宅後レビュー中に次回候補探し


- その回だけの記録は個別Planへ

## 8. 自動判定とAI/APIの扱い

### 8.1 API値は候補

APIは補助。主役ではない。

```text
API値：候補
ユーザー保存値：正
```

禁止:

- API失敗で保存不可
- API値で保存値を勝手に上書き
- APIがないと画面が崩れる

### 8.2 AIの役割

AIは自動確定者ではなく整理補助。

例:

```text
この写真は「コタ場内散歩」と「木陰スポット」に関係しそう
この音声メモは「次回改善」にもリンクできそう
```

ユーザーに対しては、候補提示まで。  
勝手な確定は禁止。

### 8.3 信頼度

すべての自動紐付けは信頼度を持つ。

```text
manual: ユーザーが指定
autoHigh: GPS/時間/画面文脈から高信頼
autoLow: 推測候補
unknown: 未確認箱
```

- 話す

### 9.3 毎回確認しない

禁止:

```text

階層を毎回選ばせる
```

正:

```text

あとでまとめて整理
```

### 9.4 必要なものを隠しすぎない

散歩中でもキャンプ準備を見る。  
キャンプ中でも次回候補を見る。  
ただし、保存先は勝手に変えない。

## 16. 最終LOCK文

OUTBASEは、単一プランを切り替えるアプリではない。  
OUTBASEは、複数の外出・準備・現地行動・記録・整理・改善が同時に動く外出OSである。

画面では、複雑さを見せない。  
内部では、可変階層と複数関連リンクを許容する。

表示対象と記録対象を分離する。  
表示切替で保存先を変えない。  
記録は止めない。  
不明なら未確認箱へ逃がす。  
あとから整理・付け替え・複数リンクできる。

孫まで固定にしない。  
UIは最大3階層程度に抑える。  
データは将来のAPI、AI整理、写真/動画保存、複数端末、家族共有に耐える構造にする。

04.8のルートAPIなし版は維持する。  
04.13以降は、このContext Model Lockに反する実装を禁止する。
