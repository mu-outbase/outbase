# OUTBASE Library-10A 横断監査報告

## 監査対象
- Library-10 実装一式
- Plan / Prep / Record / 資産ライブラリのローカル保存キー
- ギア / セット / カスタム / 履歴 / 車両 / ペット / 保管場所
- JSONバックアップ / 復元
- PWAキャッシュ識別子 / manifest

## 今回の監査で発見し修正した重大事項

### 1. 資産履歴がJSONバックアップ対象外
Library-10の書出し対象に `assetEvents` が含まれておらず、購入・使用・交換・修理・通院・ワクチン等の履歴が復元時に消える状態だった。

**修正**
- `events` をJSON書出しへ追加
- JSON読込時に `assetEvents` を復元
- 読込後に全資産領域をlocalStorageへ明示保存
- バックアップ版を `Library10A` へ更新

### 2. manifestの起動URLと配色が旧Library-07のまま
Library-10 ZIP内のmanifestが旧版識別子・旧テーマ色を保持していた。

**修正**
- start_urlをLibrary-10Aへ更新
- theme/backgroundを現在の白系UIへ統一
- index / service-worker / 画面遷移URLの識別子をLibrary-10Aへ統一

### 3. ペット名変更が予定同行者候補へ同期されない
ペット登録時は同行者候補へ追加されるが、名前変更時に旧名が残る状態だった。

**修正**
- ペット名変更時に予定同行者候補も更新
- ペット削除時、同名ペットが他にいなければ同行者候補から削除

### 4. 車両・ペット削除時に履歴や関連参照が残る
車両・ペットを削除しても、対象履歴やギア側の関連が残る可能性があった。

**修正**
- 車両削除時に対象履歴とギア関連を整理
- ペット削除時に対象履歴を整理
- 保管場所は登録品または関連参照がある場合、削除不可
