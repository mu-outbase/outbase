# OUTBASE_RESTORE04.15 FIELD03正ベース 外出属性UX・デザイン復旧 実装メモ

作成日時: 2026-07-10
VERSION: `outbase-restore04-15-field03-ux-restore-outing-attrs-20260710`

## 判定
- 04.14 UIは不合格扱いで不採用
- 実装ベースは04.13（Context Model最小導入）
- 見た目とルートの基準は04.8
- 外出タイプ網羅LOCK v1を反映

## 実装内容
- Context Modelは内部保持のまま、常時大型説明カードを出さない
- 外出属性モデル `outingAttributes` / `outingTemplates` を追加
- 通常散歩 / ドライブ散歩 / キャンプ / イベント同伴あり / イベント同伴なし / 予定なし記録を最小シナリオとして入口化
- ルートはキャンプ専用ではなく外出共通エンジンとして扱う
- ルート画面は04.8の一画面性を維持
- タブホームは説明ではなく入口として再構成
- 表示中/保存先は小型ステータスのみ表示

## 破棄したもの
- 04.14の大型Context説明カード
- 04.14の重いタブホーム説明構成
- ペットイベント=ペット同伴の決め打ち
- 散歩=自宅から歩くだけの決め打ち

## 実機確認ポイント
1. ルートが04.8のように一画面で見やすいか
2. ホーム/準備/記録が入口として迷わないか
3. ドライブ散歩と通常散歩を分けて開始できるか
4. ペットイベント同伴なしを作れるか
5. 表示切替で保存先が勝手に変わらないか
