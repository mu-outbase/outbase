# OUTBASE Library Implementation LOCK v1.4

## UI
- 資産画面を下部ナビの上に表示し、下部5タブを常時操作可能にする。
- ヘッダー・上段タブ・ギア内タブを圧縮する。
- タブ切替時は新しいタブの先頭から表示し、以前の別タブ位置を持ち込まない。
- カテゴリチップは白背景・濃色文字、選択中は深緑背景とする。

## 組合せ
ユーザーが迷わない基本関係を優先する。
1. 必ず一緒
2. 条件で一緒
3. 取付・装着
4. 対応する
5. 燃料
6. 収納先

対象はギア／車・自転車／保管場所を選択できる。
条件は、常時、インナーテント使用時、寒い時、厳寒時、雨天時、電源なし、施設設備なし、薪ストーブ使用時を保存できる。

## セット
- 同じギアは複数セットへ所属可能。
- セット内の各ギアを必須／任意へ切替可能。
- 使用条件を自由入力可能。
- 将来の予定・天気・施設条件による自動候補判定用データとして保持する。

## 入力支援
- 名称からカテゴリ・役割・代表タグを簡易補完する。
- タグは候補チップと自由入力を併用する。
- 購入単価と数量から小計を即時表示する。
- 資産ヘッダーへ登録価格合計を表示する。

## 未実装境界
保存した使用条件を、外部天気や施設情報と照合して自動的に今回の予定へ追加する処理は未接続。Library-09では条件データを登録・保持できる段階までをLOCKする。
