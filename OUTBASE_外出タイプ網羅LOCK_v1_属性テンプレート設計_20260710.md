# OUTBASE 外出タイプ網羅LOCK v1
## 属性テンプレート設計 / 04.15実装前固定

作成日: 2026-07-10  
位置づけ: RESTORE04.15実装前LOCK  
結論: **外出タイプを固定列挙だけで実装しない。外出プラン + 属性 + セッション + 記録 + 関連リンクで吸収する。**

---

## 0. 今回の反省からの固定

04.14までの失敗は、機能・導線・画面・デザインを十分に監査しないままZIP化したこと。  
以後、OUTBASEでは次を固定する。

- ZIP作成は最後。
- 実装前に、外出タイプ・属性・画面入口・保存先・将来拡張を監査する。
- ユーザーが指摘してから外出タイプを増やす後追い運用は禁止。
- `node --check OK` は合格ではない。
- 「使いたい」「迷わない」「格好いい」「将来壊れない」まで見てから実装する。

---

## 1. OUTBASEの定義

OUTBASEは、キャンプ専用アプリでも、散歩専用アプリでも、ペット同伴前提アプリでもない。

```text
OUTBASE = 外出OS
```

対象は、家族・ペット・キャンプ・散歩・イベント・買い物・下見・記録・整理・改善を含む。

---

## 2. 外出タイプは固定分類ではなくテンプレート

外出タイプを固定メニューだけで作ると、次のように必ず漏れる。

- 自宅からの通常散歩
- 車で移動してからのドライブ散歩
- キャンプ場内散歩
- ペットイベント同伴あり
- ペットイベント同伴なし
- Snow Peakイベント
- 買い物だけ
- 下見だけ
- 予約前調査
- 中止・延期・撤退
- 帰宅後片付け
- ギア乾燥
- Instagram / note投稿整理

そのため、OUTBASEでは次の構造にする。

```text
外出プラン
├ 外出テンプレート
├ 属性
├ セッション
├ 記録素材
└ 関連リンク
```

## 4. 外出属性

外出タイプだけでは判断しない。  
表示する入口は、属性の組み合わせで決める。

```text
templateType: 散歩 / キャンプ / イベント / 日帰り / 下見 / 買い物 / 用事 / 整理
stayType: 宿泊あり / 日帰り / 仮眠 / 滞在なし
transport: 徒歩 / 車 / 電車 / 複合 / 未定
petRelated: あり / なし
petAccompanied: あり / なし / 未定
targetPets: コタ / アオ / エラ / ユキ / 複数 / なし / 未定
companions: ひとり / 夫婦 / 友人 / 家族 / グループ / 未定
reservation: あり / なし / 未定
ticket: あり / なし / 未定
weatherImpact: 高 / 中 / 低
routeImportance: 高 / 中 / 低
gearImportance: 高 / 中 / 低
foodImportance: 高 / 中 / 低
recordPurpose: 思い出 / 下見 / 買い物 / 改善 / 体調 / ギア / 投稿 / 比較
status: 候補 / 予定 / 準備中 / 当日 / 実行中 / 帰宅後 / 中止 / 延期 / 完了
```

## 5. ペット関連イベントの固定

「ペットイベント = ペットを連れて行く」と決め打ちしない。

```text
外出プラン: イベント参加
イベント種別: ペット関連イベント
ペット同伴: あり / なし / 未定
```

### 同伴ありで出す入口

- ルート
- 駐車場
- 入場時間
- 持ち物
- 水 / 休憩
- 暑さ寒さ
- カート
- 会場内散歩
- 写真
- 買い物
- メモ
- 体調メモ

### 同伴なしで出す入口

- ルート
- 駐車場
- 入場時間
- 見たいブース
- 買いたい物
- 気になる商品
- 写真
- メモ
- 次回ペット同伴できるか
- 家族へ共有する情報

### 未定で出す入口

- 同伴可否確認
- 会場ルール
- 駐車場
- 混雑
- 暑さ寒さ
- カート可否
- 同伴する / しない の判断メモ

## 6. ルートはキャンプ専用ではない

ルートはキャンプ準備の一部ではなく、外出共通エンジン。

対象:

- キャンプ場へ行く
- 公園へドライブ散歩に行く
- ペット関連イベントへ行く
- Snow Peakイベントへ行く
- ドッグカフェへ行く
- 道の駅へ行く
- 下見へ行く
- 病院 / トリミングへ行く

ただし、表示内容は外出属性で変える。

### キャンプ

- チェックイン逆算
- 移動時間
- 給油
- コンビニ
- 買い物
- SA/PA
- 到着余裕
- 出発時刻

### ドライブ散歩

- 到着予定
- 駐車場
- 散歩開始
- 滞在時間
- 帰宅目安
- 水 / 休憩

### イベント参加

- 入場時間
- 駐車場

```text

├ Session: 往路ルート
├ Session: 買い物
├ Session: 設営
├ Session: 料理
├ Session: コタ場内散歩
├ Session: 撤収
└ Session: 帰宅後レビュー
```

```text
Plan: 手賀沼ドライブ散歩
├ Session: 往路ドライブ
├ Session: 現地散歩
├ Session: 休憩
└ Session: 復路ドライブ
```

```text
Plan: ペット関連イベント 同伴なし
├ Session: 会場ルート
├ Session: 物販チェック
├ Session: 気になる商品記録
├ Session: 次回コタ同伴可否メモ
└ Session: 帰宅後整理
```

## 8. 記録素材と関連リンク

記録素材は1つの親だけに固定しない。

```text
Record: 写真A
rootPlanId: ペットイベント
sessionId: 物販チェック
links:
  - 気になる商品
  - 次回キャンプギア候補
  - コタ用品候補
```

```text
Record: 音声メモB
rootPlanId: 赤城山キャンプ
sessionId: コタ場内散歩
links:
  - 危険ポイント
  - 次回改善
  - 場所メモ
```

固定:

- 記録は recordTarget に保存する
- 表示中プランを変えても保存先は勝手に変えない
- 不明時だけ未確認箱へ入れる
- 後から複数リンクできる
- 1つの記録がキャンプ・散歩・商品・改善にまたがってよい

## 9. UIに出すもの / 出さないもの

### 常時出してよい

- OUTBASEロゴ
- 小型の表示中
- 小型の保存先
- 下バー: 予定 / 探す / 準備 / 記録 / 思い出

### 常時出さない

- Context Model説明
- 親子孫ツリー
- 全テンプレート一覧
- 全属性一覧
- 大型ステージバー
- 支払い確認などMVP外ノイズ
- 長い説明カード

### ホームは説明画面ではなく入口画面

```text
予定 → 次の外出 / 今日 / 追加 / 切替
探す → キャンプ場 / 散歩場所 / イベント / カフェ / 下見 / 保存候補
準備 → ルート / 持ち物 / 買い物 / 料理 / ギア / 天気
記録 → 話す / 撮る / 動画 / 場所 / ピン / メモ / 散歩開始
思い出 → 記録一覧 / 写真・動画 / レビュー / 次回改善 / 場所メモ
```

ただし、各入口の表示優先度は外出属性で変える。

## 10. 実装前の合格条件

04.15実装前に以下を満たすこと。

### 設計

- 外出タイプを固定列挙だけで処理していない
- 外出属性で表示入口を変えられる
- ペット関連イベントがペット同伴前提になっていない
- ルートがキャンプ専用になっていない
- 通常散歩 / ドライブ散歩 / 場内散歩を分けられる
- キャンプ / デイキャンプ / 宿泊 / イベント / 下見 / 買い物 / 用事 / 帰宅後整理を吸収できる

### 保存

- 表示中と保存先が分離している
- recordTarget に保存できる
- sessionId を持てる
- links を持てる
- 未確認箱は最後の保険

### UI

- ホームが説明画面になっていない
- Context Modelが画面に出しゃばらない
- 04.8ルートの一画面性を壊さない
- 見た瞬間に使いたい入口が分かる
- デザインがOUTBASEらしい
- スマホで押しやすい

### 監査

- 04.8の良さを守っている
- 04.14の悪化を再発していない
- 追加外出タイプが来ても設計が壊れない
- ZIP作成前にワイヤーで確認する

## 12. 最終固定

```text
OUTBASEは外出タイプを固定メニューで閉じない。
外出プランに属性を持たせ、属性に応じて入口を出し分ける。
キャンプ専用、散歩専用、ペット同伴前提にはしない。
Context Modelは内部に残し、画面では小さく表示中/保存先だけ見せる。
ルートは外出共通エンジンとして扱い、04.8の一画面性を壊さない。
```
