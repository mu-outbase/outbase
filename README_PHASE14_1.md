# OUTBASE Phase14.1 複数プラン分離修正

## 修正理由
Phase14のレビュー・候補・参照記録が主役プラン別に分離されておらず、
複数プラン同時進行時に別プランの内容が混ざる可能性がありました。

## 修正内容
- レビューを主役プランIDごとに保存・表示
- 候補を主役プランIDごとに保存・表示
- 保存済み記録を現在の主役プランに限定
- 候補採用時も元のプランIDを保持
- 主役プラン切替を検知し、思い出画面を自動更新
- プラン未選択時は「共通／未割当」として分離
- 既存Phase14の未紐付けデータは、プラン未選択時だけ表示

## 守る条件
- v142 非直線エントリー維持
- 複数プラン同時進行維持
- FIELD03正ベース維持
- AIは提案のみ
- 候補は採用まで保存先へ反映しない
