# OUTBASE MVP固定版 v140

## 固定結論

MVPα1は凍結する。これ以上、1ファイル `index.html` に機能を足さない。

OUTBASE MVPは、機能数を減らした版ではない。むーが実際に使いたくなる最低完成ラインである。

## MVP成立ライン

1. ホームを開く理由がある。
2. 次のキャンプ準備が、手入力ではなく取込・候補・承認で立ち上がる。
3. 予約スクショ読取だけで終わらず、買い物、持ち物、コタ用品、前回反省、リン送信用リストへつながる。
4. 当日は散歩、写真、動画、音声文字起こし、GPS、場所カードを3秒以内で残せる。
5. 帰宅後にレビューし、次回改善へ返る。
6. PWAとしてスマホホーム画面から開ける。
7. 自動保存、未終了復元、オフラインを前提にする。

## MVPではやらないこと

- 手入力中心の予定管理
- 予約スクショ読取だけで完了扱い
- 巨大 `index.html` への追加実装
- Codexに思想や優先順位を丸投げ
- いきなりネイティブアプリ化

## 本開発の中心体験

```text
予約情報 / スクショ / PDF / メール / カレンダー
↓
次のキャンププロジェクト
↓
準備候補：持ち物・買い物・コタ用品・前回反省
↓
当日記録：散歩・写真・動画・音声文字起こし・GPS
↓
レビュー：忘れ物・設営撤収・料理・天気判断
↓
次回改善
```
