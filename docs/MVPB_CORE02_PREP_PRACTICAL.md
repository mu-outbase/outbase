# OUTBASE MVPβ Core02 — 準備候補の実用化

## 目的

Core01で確認した「予約情報 → 候補抽出 → 次のキャンプカード → 準備候補 → リン送信用リスト」の流れを、実際にキャンプ前に使える準備候補へ育てる。

## Core02で追加するもの

- 準備条件入力：天気、最高気温、最低気温、降水、風、人数、コタ同行、献立、過去反省、ギアメモ。
- 実用準備候補：泊数、人数、暑さ寒さ、雨、風、コタ同行、献立、ギアメモから候補を増減する。
- LINE貼付けリスト改善：条件と候補をまとめてリンに送れる形にする。
- UX01保留分の一部吸収：スクショ取消、取込内容リセット、保存後の準備画面反映。

## 重要な位置付け

OUTBASEはキャンプ専用ではない。Core02は最初の強い利用シーンであるキャンプ前準備を実用化する段階。Core03で散歩・当日・日常記録へ広げる。

## 成功判定

1. 次キャンプカード作成後に準備条件欄が出る。
2. 天気・人数・コタ同行・献立・過去反省・ギアメモを入れられる。
3. 「準備候補を更新」で買い物・持ち物・コタ用品・反省が変わる。
4. リンに送るリストへ条件が反映される。
5. これにより「キャンプ前なら使ってもいい」に到達する。
