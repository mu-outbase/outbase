# OUTBASE v166.2 全画面密度再設計LOCK

## 背景

Android実機でv166.1の軽量化は合格したが、正式デザインには次の問題が残った。

- 見出しが大きすぎる
- セクション間の空白が広すぎる
- 情報量に対してカードや面が高すぎる
- 下部ナビと中央追加ボタンの主張が強すぎる
- TRAIL LENSの記録詳細シートが大きく、閉じる操作が主役になっている
- 英字ラベルが装飾として浮き、試作画面に見える

## 正式密度ルール

- 高級感を大文字・広い空白で作らない。
- 日本語の情報階層、基準線、細い罫線、正確な余白で完成感を作る。
- 主要タップ領域48pxは維持する。
- ページタイトルはスマートフォンで原則31〜38px以内。
- セクション見出しは21px基準。
- 一覧行は62〜70px基準。
- 画面下部ナビは64px基準。
- ボトムシートは最大74dvh。内容に応じて自然に縮む。
- 装飾目的だけの英字eyebrowは廃止する。
- FIELD03処理を変更せず、CSSと新シェル表示層だけで実現する。

## NORTH

- ホームの日付を29〜36pxへ縮小。
- 「今・次・すぐ使う・最近」の間隔を約30%削減。
- 次の活動カードを17〜18px paddingへ縮小。
- すぐ使うは4列を維持し、スマートフォンでは補助説明を隠す。
- 探す・保管庫のページタイトルを31〜38pxへ縮小。
- 活動詳細のタイトル、進行レール、事実グリッド、一覧行を再調整。
- カレンダーの上下余白と日セル高さを縮小。
- 下部ナビは中央ボタンを含めて同じ高さへ揃える。

## TRAIL LENS

- 記録ヘッダー、地図カード、操作パネルの余白と影を縮小。
- 地図は最大43dvh／350pxを基準にし、操作領域を同時に見せる。
- 記録詳細シートは最大74dvh、背景blurなし。
- シート見出しは20〜25px。
- 情報行を48px基準へ統一。
- 補助操作と閉じる操作は48pxを維持しながら視覚的な主張を抑える。

## 禁止

- 5タブへ戻さない
- 大型グローバルバー・大型chipを追加しない
- 全画面overflow:hiddenを使わない
- MutationObserverを追加しない
- FIELD03のGPS・写真・音声・保存処理を変更しない
- 密度改善を理由に48pxタップ領域を壊さない
