# OUTBASE BOOT v9.2 / Core08-E3 正本統合MVP反映

## 位置づけ
E2は「全部入り」と言いながら、正本の思想をUIに変換できず、ボタンの羅列になっていた。
Core08-E3では、MASTER_v139 / MVP監査LOCK / v158までの反省を見直し、OUTBASEの本質である「予定 → 準備 → 当日 → 現地記録 → 思い出整理 → 次回改善」を、＋記録入口と未確認箱に統合する。

## 正本から読み直した必須思想
- OUTBASEは単なる記録アプリではなく、体験を資産化するアプリ。
- 入力は最小、記録は最大、AI/OUTBASEは裏で補完する。
- 記録中は分類させない。写真・動画・音声・メモに集中する。
- ユーザーは紐付けしない。時間/GPS/写真/音声/メモ/予定から候補化する。
- 未分類・未確認だけを後で確認する。
- 自宅散歩とキャンプ場散歩は別物。
- 散歩は地図/GPSが主役。
- Google Mapsを採用し、ドライブ中はOUTBASEを操作させない。
- キャンプ・散歩・旅行・ピクニックはイベント種別として扱う。
- 事実保存とAI生成物を分離する。
- Google Photosは写真/動画の正本。OUTBASEは管理・検索・再生成。
- 間違い登録、終了、破棄は復旧できる前提。

## Core08-E3で反映した内容
### 1. ＋を「操作ボタン集」から「今日の運転席＋記録入口」へ変更
- 今日の運転席：出発前 / 往路 / 到着受付 / 設営 / 滞在 / 撤収 / 帰路 / 帰宅後
- 今すぐ残す：写真 / 声 / メモ / あとで
- 活動を始める：散歩 / ドライブ / 設営 / 料理 / 撤収 / 場内探索
- 準備から引き継ぐ：予定 / ルート / 料理 / ギア / コタ / 天気 / 買い物 / 設営

### 2. 活動開始後の画面を整理
- 上部：活動名、予定紐付け、経過時間、距離または記録数
- 記録：写真 / 声 / メモ / あとで
- 活動別候補：ドライブ、設営、料理、撤収、散歩、場内探索で出し分け
- 終了：保存して未確認箱へ
- 切替：必要時のみ開く

### 3. ドライブモード
- Google Mapsで開く
- ドライブ中はOUTBASEを操作しない
- 給油 / 買い出し / 休憩 / 渋滞 / コタ休憩 / 現在地を候補化
- 戻った後、未確認箱で整理

### 4. 散歩・場内探索
- GPS取得と距離保存
- 自宅散歩と場内探索を分離
- 場内探索は設備・景色・注意・コタ・天気を候補化

### 5. 設営・料理・撤収
- 設営：レイアウト / テント / タープ / 寝室 / 外回り / 注意 / 完了
- 料理：量 / 味 / 余り / 次回料理 / 写真 / 注意
- 撤収：濡れ物 / 忘れ物 / 収納 / コタ / 天気 / 完了

### 6. 裏側データ
- recordHistory：未確認箱表示用
- dayRecords：当日候補保存
- dayAutoEvents：フロー/活動/Google Maps/GPS候補
- reviewQueue：次回改善・ギア・料理・コタ・天気などのタグを蓄積
- context：予定・準備・活動種別・正本参照を保持

### 7. テストリセット・復元
- ＋由来のテスト記録のみリセット
- 予定・準備・ギア・本番記録は消さない
- 直前状態は quickResetBackup に残し復元可能

## 重要なLOCK
E3以降、＋は「ボタンを並べる場所」ではない。
＋は、正本の思想を現地で使える形に落とした「行動入口」。

画面に全部は出さないが、裏では必ずつなぐ。
- 予定
- 準備
- ルート
- Google Maps
- 写真/動画/声/メモ
- GPS
- 散歩/ドライブ/設営/料理/撤収/場内探索
- 未確認箱
- 思い出整理
- 次回改善
- チャッピー提案材料
