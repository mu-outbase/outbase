OUTBASE v166.31 / R0-02 準備表示高速化 v17.3

実機報告
- v17.2で準備白画面は解消した
- 予定詳細から準備表示まで約5秒かかった
- 表示成功だけでは合格にせず、R0継続とする

原因
1. 準備ルート表示前に共通シェルモデル全体を生成していた
2. ensureBaselineを画面描画前に完了待ちしていた
3. 準備集計とは別に予定詳細全体も読み込み、記録・メディア・レビュー・改善・全ギア等を取得していた
4. 準備集計が予定情報と準備項目を再読込し、同じデータを重複取得していた

v17.3修正
- 準備ルートでは共通シェルモデル全体を待たず、ヘッダー・本文・下ナビを即時生成
- 準備表示に必要な活動本体・日程・準備項目だけをIndexedDBから並列取得
- 予定詳細全体の読み込みを準備初期表示から除外
- 基礎準備の永続化は初期画面表示後にバックグラウンド実行
- 永続化完了後のみ準備項目を自動更新
- 30秒の準備表示キャッシュを追加

保護
- FIELD03本体変更なし
- GPS・地図・写真・動画・音声変更なし
- outbase_db / outbase_story_db定義変更なし
- HOME v36 r34天気変更なし
- カレンダー変更なし
- 予定詳細v16.3変更なし
- GitHub main未変更

確認条件
1. Androidで予定詳細→準備をタップ
2. 白画面にならない
3. 体感待ち時間が大幅に短縮する
4. 初回表示直後に準備項目が「準備中」から操作可能へ更新される場合がある
5. チェック操作、予定詳細へ戻る、活動を始める、詳細な準備を開くを確認
