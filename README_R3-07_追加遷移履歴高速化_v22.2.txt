OUTBASE R3-07
追加メニュー遷移・戻る履歴・表示待ち修正 v22.2

【実機報告から確認した問題】
・＋追加の全項目が、開いていたカレンダーへ戻ってしまう
・活動開始と詳細な準備が、押してから画面反応まで遅い
・戻る操作を2回行わないと元画面へ戻れない
・探す画面から各項目を開くと、画面反応まで時間がかかる

【原因】
1. 追加シートを閉じる際に history.back() を実行した直後、新ルートを追加していた。
   非同期popstateと新ルート遷移が競合し、カレンダー等の元画面へ戻っていた。
2. モーダル用履歴を残したまま新画面をpushしていたため、戻る履歴が二重になった。
3. 新ルートはデータ取得完了まで内容を描画せず、端末上で「押しても動かない」ように見えていた。
4. 詳細準備は基準項目の永続化を表示前に待ち、開始・メモ・場所は全予定走査を毎回行っていた。
5. 探す・保管庫から予定詳細へ進む際、詳細データの先読みが不足していた。

【v22.2の修正】
・＋追加の選択時は、モーダル履歴をhistory.backせず閉じる。
・モーダル履歴エントリを選択先ルートでreplaceし、戻る1回で元画面へ戻す。
・ルート履歴へ残っていた outbaseModal 状態を除去する。
・同一URLへの重複pushを防止する。
・各新ルートで、データ取得前に画面本体を即時描画する。
・入力中に裏側の読み込みが完了しても、入力値とスクロール位置を保持する。
・詳細準備は既存キャッシュを優先し、基準項目永続化を表示後へ移動する。
・活動開始、メモ、場所はHOME／保管庫の既存キャッシュを優先し、全予定走査を回避する。
・＋追加を開いた時点で活動開始・メモ・持ち物を先読みする。
・探す／保管庫では、表示中の予定詳細と持ち物・場所を先読みする。
・予定詳細、探す、保管庫にも即時の読み込み表示を追加する。
・各新画面へ returnShell を渡し、画面内の戻るは明示された元画面へ1回で戻る。

【保護範囲】
・現行HOMEの構造と天気表示は変更していない。
・旧FIELD03本体は変更していない。
・GPS、地図、写真、動画、音声、場所ピン、駐車位置、Wake Lock、IndexedDB、オフライン復元は変更していない。
・GitHub mainへの書き込みは行っていない。
