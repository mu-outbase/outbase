## カレンダー正式動作 / 土日祝 / 連動日程バー / スクロール保持 / 散歩地図 / 04.8ルート保護

# 0. 結論

RESTORE04.17は方向性として、

- トップをカレンダー主役に戻す
- 外出タイプ選択をトップから下げる
- 記録タブに散歩マップを戻す
- ルートと散歩地図を分ける

という点は正しい。

しかし、実機指摘と全体再監査により、04.17は以下が未合格。

```text
・カレンダー動作が過去に決めた正式仕様と違う
・連動日程 / 連泊予定の表示が違う
・土日祝が分かりにくい
・日付タップや画面内操作のたびに上へ戻る
・カレンダー、散歩地図、ルート、タブ導線の回帰検査が不足していた
```

04.18は機能追加版ではない。  
**カレンダー正式動作復元 + スクロール不具合修正 + 既存正本保護版**とする。

# 3. カレンダー正式動作LOCK

## 3-1. トップの主役

予定タブ / HOMEの主役は、外出タイプ一覧ではなくカレンダー。

正しいトップ:

```text
今週 / 今月カレンダー
今日の予定
選択日の予定
次の外出
未確認記録
＋新規予定
```

禁止:

```text
・トップにキャンプ / 通常散歩 / ドライブ散歩 / イベント同伴ありなしを大きく並べる
・外出タイプ選択をトップ画面の主役にする
・カレンダーより属性テンプレートを目立たせる
```

外出タイプ選択は、**＋新規予定** の中に下げる。

---

## 3-2. 日付タップ動作

カレンダー日付の動作は以下で固定。

```text
1回目タップ:
  その日を選択し、予定 / ToDo / メモを表示する。
  画面上部へ戻らない。

同じ日付をもう1回タップ:
  予定追加パネルを開く。
  画面上部へ戻らない。

別の日付をタップ:
  選択日を切り替え、下部の予定 / ToDo / メモを更新する。
  画面上部へ戻らない。
```

今日へ戻る操作:

```text
カレンダー内の「今日」ボタンに統一。
下バーやHOMEの別導線で勝手に今日へ戻さない。
```

---

## 3-3. 選択日の表示

選択日の下には以下を出す。

```text
・その日の予定
・その日のToDo
・その日のメモ
・連泊 / 連続予定の中での位置
・未確認記録がある場合は控えめに表示
```

例:

```text
7/18（土）
スノーピーク赤城山CF 初日
・出発
・チェックイン
・設営
・夜ごはん
```

```text
7/19（日）
スノーピーク赤城山CF 滞在日
・朝ごはん
・場内散歩
・写真整理
```

```text
7/20（祝）
スノーピーク赤城山CF 最終日
・撤収
・帰宅
・ギア乾燥
```

# 4. 土日祝表示LOCK

## 4-1. 色だけに頼らない

土日祝は色だけで表現しない。

必須表示:

```text
・土曜: 「土」表記
・日曜: 「日」表記
・祝日: 「祝」表記
・祝日名: 表示可能なら表示
・振替休日: 祝日扱い
```

推奨:

```text
・土曜: 青系または寒色ニュアンス
・日曜: 赤系または暖色ニュアンス
・祝日: 金赤系 / 祝マーク
・連休: 背景または予定バーで連続感を出す
```

例:

```text
7/11 土
7/12 日
7/20 祝 海の日
```

## 4-2. 祝日データ

MVPでは固定簡易祝日テーブルでよい。
API連携は後工程。

必要最低限:

```text
・祝日判定関数
・祝日名取得関数
・土日祝class付与
・祝日名の短縮表示
```

# 5. 連動日程 / 連泊予定LOCK

## 5-1. 連泊予定は日ごとにバラさない

キャンプや宿泊など、開始日と終了日を持つ予定は、カレンダー上で繋がるバー表示にする。

```text
7/18 ━ 7/19 ━ 7/20
スノーピーク赤城山CF
```

禁止:

```text
・各日に同じ予定カードを別々に大量表示する
・連泊なのに初日/中日/最終日の区別がない
・連動日程が分からない
```

---

## 5-2. 日程内の役割表示

連続予定は選択日ごとに役割を出す。

```text
初日: 出発 / 到着 / チェックイン / 設営
中日: 滞在 / 料理 / 散歩 / 記録
最終日: 撤収 / チェックアウト / 帰宅 / 片付け
```

# 6. スクロール位置保持LOCK

## 6-1. 不具合判定

次の挙動は不具合。

```text
・日付タップごとに上へ戻る
・予定カードタップごとに上へ戻る
・詳細開閉ごとに上へ戻る
・保存先切替ごとに上へ戻る
・表示対象切替ごとに上へ戻る
・同じタブ内の操作で毎回先頭へ戻る
```

## 6-2. 正しい挙動

```text
タブ切替:
  先頭へ戻ってよい。

同じタブ内の日付タップ:
  スクロール位置を維持。

同じタブ内のカード開閉:
  スクロール位置を維持。

予定追加パネル表示:
  必要ならパネル位置へ軽く寄せるが、勝手に最上部へ戻さない。

記録タブ内の操作:
  散歩マップ位置やボタン位置を維持。
```

## 6-3. 実装方針

```text
・render前に scrollY を保存
・render後に必要な場合だけ復元
・タブ切替時のみ明示的に先頭へ戻す
・日付選択や詳細開閉では scrollTo(0,0) を呼ばない
・旧関数内の無条件 scrollTo を監査して削除 / 条件化する
```

現コードは `scrollTo` 系が複数存在するため、04.18ではここを重点監査する。

# 7. 散歩地図LOCK

## 7-1. 地図は2種類に分ける

```text
移動ルート地図:
  車で目的地へ行くための地図。
  キャンプ場、公園、イベント会場、ドッグカフェなどへ向かう。

現地 / 散歩地図:
  実際に歩いている場所の地図。
  GPSログ、ピン、写真、音声、危険ポイント、休憩場所を残す。
```

## 7-2. 記録タブの必須入口

記録タブには以下を分かる位置に出す。

```text
・散歩マップ
・散歩開始
・現在地
・ピン
・写真
・音声
・メモ
・Google Map
```

`場所` という曖昧なボタンだけでは不可。
