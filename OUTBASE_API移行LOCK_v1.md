# OUTBASE API移行LOCK v1

## 位置づけ
RESTORE04系はAPIなしMVPを安定させる段階。
ただし、将来APIありへ移行する前提を保持する。

## 固定方針
- OUTBASEはAPIなしで成立する。
- APIありでは精度と自動化を強化する。
- APIは裏側の補助であり、画面主役や必須条件にしない。
- API値は最初は候補扱い。
- ユーザー保存値を勝手に上書きしない。
- API失敗・未設定・通信不可でも、手入力値・保存値・Google Maps起動で画面を維持する。

## ルートAPI移行時
1. Google Maps等から移動時間候補を取得。
2. 保存済み移動時間とAPI候補を並べる。
3. 採用する / 今回だけ使う / 無視 をユーザーに選ばせる。
4. 採用後にだけ保存値へ反映する。

## 禁止
- API必須アプリにしない。
- 取得失敗で保存・表示・現地記録を壊さない。
- ルート画面を地図だらけに戻さない。
- キャンプ場住所の手入力前提に戻さない。
- 現地セッション/GPS/地図/写真/動画/音声/ピン/欠測/画面ON維持を壊さない。
