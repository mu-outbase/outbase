# OUTBASE LOCK v146 既存UI維持・全予定カレンダー拡張設計LOCK

作成日：2026-07-07

## 結論
v146は実装ZIPではない。
復旧した RESTART-35 / Restart-37〜40系の画面を正ベースとして固定し、
その既存UIを壊さずに「全予定カレンダー」「非直線エントリー」「文脈仮紐付け」を入れるための設計LOCKである。

## 正ベース
現在の正ベースは以下。

- OUTBASE / RESTART-35
- ホーム見出し：今日は何する？
- 高機能カレンダー
- 小型プランチップ
- 下ナビ：予定 / 探す / 準備 / ＋ / 思い出
- 37〜40系の色・余白・カード・ボトムナビ

この画面構造を壊さない。

## v144 / v145 の失敗原因
v144/v145は、全予定カレンダー統合の考え方は近づいたが、
実装で新しいカレンダーUI・新しいホーム・新しいカード構造を作ってしまった。
これは「既存UI維持」ではない。

今後は禁止。

- 新しいホームを作る
- 新しいカレンダー画面を作る
- 「全予定 / 37-40復旧ベース」などの新ラベルを出す
- カードを増やす
- 巨大カレンダー化
- CSSだけ似せて復旧と呼ぶ
- app.jsを別アプリとして書き換える

## v146の基本方針
見た目を足さない。
既存画面の意味を拡張する。

- 予定 = 全予定カレンダー
- 探す = 候補探し・キャンプ場/場所/外出先探し
- 準備 = 選択中の予定/出来事に紐づく準備
- ＋ = 予定/準備/現地/実績/メモを追加する入口
- 思い出 = 過去実績・記録・整理・改善

## 既存画面に何を足すか
### ホーム
変更しない。
「今日は何する？」と高機能カレンダーを正とする。
今日・次・要確認の情報を出す場合も、既存カードの範囲内で出す。
新しいヒーローや巨大ボタンは禁止。

### 予定
既存の高機能カレンダーを全予定対応にする。
キャンプ予定だけではなく、仕事・病院・支払い・家族・ペット・買い物・車・散歩・旅行なども入れる。
見た目は既存カレンダーを維持する。

### 探す
予定ではなく候補探しの場所。
キャンプ場、外出先、散歩先、買い物先、イベント候補などを扱う。
ジョルテ代替カレンダーとは混ぜない。

### 準備
現在選択中の予定/出来事に紐づく準備を扱う。
予定がなくても準備から始められる。
ただし、作成後は文脈で予定/出来事に仮紐付けする。

### ＋
既存の＋入口を維持する。
中身は5入口にする。

- 予定
- 準備
- 現地記録
- 過去実績
- メモ

押したら単なるタブ移動だけは禁止。
最低限の保存または作成まで行う。

### 思い出
過去実績、記録、整理、改善を扱う。
キャンプ以外の外出や散歩も思い出化できる。
ただし、全件整理箱にしない。

## データ設計
基本単位は「予定/出来事」。

予定/出来事は以下を持つ。

- 予定名
- 種別
- 深さ
- 日時
- 場所
- メモ
- 準備
- 現地記録
- 過去実績情報
- 思い出
- 改善
- 文脈候補

## 予定の深さ
- Lv.1 普通の予定：仕事、病院、支払い、美容室など
- Lv.2 メモ/持ち物あり：買い物、車点検、ペット病院など
- Lv.3 準備あり：外出、イベント、散歩遠征、旅行など
- Lv.4 記録/整理/改善あり：キャンプ、旅行、犬連れ外出など

全予定を重くしない。
深い予定だけ詳細展開する。

## 非直線エントリー
入口は1つに決めない。

- 予定から始まる
- 準備から始まる
- 現地から始まる
- 過去実績から始まる
- メモから始まる

どこから始まっても、文脈で予定/出来事に仮紐付けする。

## 未整理・要確認の扱い
未整理に全部入れない。
要確認は、文脈判定で迷った少数だけを見る場所。

保存時にまずOUTBASEが以下を使って仮紐付けする。

- 日付
- 時刻
- 場所
- 直近予定
- 開催中予定/出来事
- 入力内容
- 押した入口
- 前後の記録

ユーザーに全件整理を押し付けない。

## Googleカレンダー / ジョルテ移行
v146では設計のみ。
Googleカレンダー本接続は後工程。
ジョルテ独自データ直接読込は前提にしない。

目標は、最終的にジョルテを使わずOUTBASEで全予定を見ること。
ただし、初期実装はローカル予定から開始する。

## 次の実装方針
次に作る実装ZIPは v147 とする。

v147は、
「既存UI維持・全予定カレンダー拡張 一括実装ZIP」。

条件：
- 37〜40の画面構造を維持
- 新ホーム禁止
- 新カレンダーUI禁止
- v144/v145コード残り禁止
- index.html / style.css / src/app.js / manifest.json / service-worker.js を一括で整合
- CLEAN復旧のような混在回避を必須とする

## 完成判定
v147の合格条件。

- 開いた瞬間、RESTART-35 / 37〜40系に見える
- 下ナビが予定 / 探す / 準備 / ＋ / 思い出のまま
- 高機能カレンダーが全予定を扱える
- 普通の予定を追加できる
- 準備から始められる
- 現地記録から始められる
- 過去実績から始められる
- 要確認が膨大な未整理箱にならない
- v144/v145のラベルや画面構造が残っていない
