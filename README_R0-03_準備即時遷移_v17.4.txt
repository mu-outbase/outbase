OUTBASE R0-03 準備即時遷移 v17.4

対象
- 予定詳細から準備を開くまで3秒以上かかる実機不具合

原因
- 準備タップ時にIndexedDBの現在活動保存完了を待ってからルーター遷移していた
- 準備ルートへ入った後も、既に予定詳細が保持している準備情報を使わず再取得していた
- 準備用シェルが先行表示済み画面を読込表示で上書きしていた

修正
- localStorageの活動文脈を同期反映後、準備へ即時遷移
- IndexedDBのruntime_context更新は遷移後のバックグラウンド処理
- 予定詳細が既に持つ予定・日程・場所・準備集計を準備画面へ事前投入
- 準備画面を同期描画し、その後にDB内容をバックグラウンド再確認
- 準備シェルが事前描画済み画面を読込表示で上書きしない
- 準備遷移ではView Transition完了待ちを使用しない

保護
- FIELD03、GPS、地図、写真、動画、音声、ピン、欠測区間、画面ON維持は変更なし
- outbase_db / outbase_story_dbの定義変更なし
- HOME v36 r34天気、カレンダー、予定詳細v16.3の表示変更なし
- GitHub mainはこのZIP生成時点では未変更

実機判定
- 予定詳細→準備の見た目切替が体感即時であること
- 画面表示後に準備件数が変化しても操作不能や白画面にならないこと
- 予定詳細へ戻り、再度準備を開いても即時であること
