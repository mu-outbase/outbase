# OUTBASE BOOT v8.8
## Core08-D8：当日シンプルハブ + テストリセット LOCK 正本反映

### LOCK日
2026-07-05

### 前提
Core08-D7は当日コネクトモードとして、間違い登録・Google Maps・モード切替・連携ハブを入れたが、画面上に情報と候補処理が出すぎた。
キャンプ中に使うには複雑で、ユーザーが整理係になっていた。

### Core08-D8の目的
当日タブは候補処理画面ではない。
当日タブは、今の状態・完了数・残件数・未確認数だけをすぐ確認し、必要な出来事を自由に残すシンプルハブとする。

### LOCK思想
- ユーザーは自由にキャンプする。
- OUTBASEは裏で出来事・変化・未確認・連携候補を拾う。
- 初期表示に候補一覧を並べない。
- 未確認は「未確認箱」にまとめる。
- 普段の操作は「残す」「違う」「あとで」だけ。
- モード切替は、今の状態が違う時だけ開く。
- 連携は表に並べず、候補として裏に残す。
- テスト中は当日データだけ安全にリセットできる。

### 画面構造
1. 今の状態カード
   - 現在モード
   - 完了数
   - 残件数
   - 未確認数
   - 終わったこと短縮表示
   - 残っていること短縮表示
   - 残す / 違う / あとで
   - ドライブ中はGoogle Maps起動

2. 自由に残す
   - メモ欄
   - 写真
   - 音声
   - メモ
   - 未分類で保存

3. 未確認箱
   - 未確認/間違い/あとで確認だけを格納
   - 普段は閉じる
   - 開いた時だけ確定/違う/あとでを処理

4. 連携サマリー
   - 準備/料理/ギア/コタ/次回への候補数だけを表示
   - 確定するまで反映しない

5. 状態修正
   - 出発前/往路ドライブ/到着受付/設営/料理/撤収/帰路/帰宅後など
   - 違う時だけ開いて切替

6. テスト中
   - 当日データだけリセット
   - リセット前バックアップを保持
   - リセット前に戻す

### テストリセット対象
- dayRecords
- dayGpsHints
- dayAutoEvents
- dayCorrections
- dayConnectQueue

- activeDayStatusStep

### テストリセットで消さないもの

- コタ情報

- 本番の過去記録
- 正本データ
