## 4. 外出属性モデルの扱い

### 4.1 外出タイプは固定メニューにしない

固定分類だけでは漏れる。

```text
通常散歩
ドライブ散歩
キャンプ
デイキャンプ
車中泊
コテージ/ロッジ
イベント参加
ペット関連イベント 同伴あり
ペット関連イベント 同伴なし
買い物
下見
病院/トリミング
帰宅後整理
予定なし記録
```

これらを全部トップ画面に出すと使いにくい。  
そのため、内部では属性で持ち、画面では必要入口だけを出す。

### 4.2 最小属性

04.15で内部に持たせる最小属性はこれ。

```text
templateType
category
stayType
transport
petRelated
petAccompanied
targetPets
companions
reservation
ticket
weatherImpact
routeImportance
gearImportance
foodImportance
recordPurpose
status
```

### 4.3 ペットイベントの固定

```text
ペットイベント = ペット同伴とは限らない
```

同伴あり:

```text
ルート / 駐車場 / 入場時間 / 持ち物 / 水 / 休憩 / 暑さ寒さ / カート / 写真 / メモ
```

同伴なし:

```text
ルート / 駐車場 / 入場時間 / 見たいブース / 買いたい物 / 気になる商品 / 写真 / メモ / 次回同伴可否
```

未定:

```text
同伴可否確認 / 会場ルール / 混雑 / 暑さ寒さ / カート可否 / 判断メモ
```

## 5. タブホーム最終方針

下バーは固定。

```text
予定　探す　準備　記録　思い出
```

### 5.1 予定

目的: 外出を選ぶ / 作る / 切り替える。

初画面:

```text
次の外出
今日
候補
追加
```

禁止:

```text
支払い確認などOUTBASE文脈外のサンプルを通常表示しない
外出属性を一覧で全部見せない
説明カードを並べない
```

### 5.2 探す

目的: 次に行く場所・候補を探す。

初画面:

```text
キャンプ場
散歩場所
イベント
お店/道の駅
保存候補
```

注意:

```text
探すは候補探し。
準備や記録の入口と混ぜない。
```

### 5.3 準備

目的: 表示中の外出へ行く前/当日に必要な準備。

初画面は属性で変える。

共通入口:

```text
ルート
持ち物
天気
メモ
```

キャンプなら追加:

```text
買い物
料理
ギア
設営
撤収
```

ドライブ散歩なら追加:

```text
駐車場
現地散歩開始
水/休憩
```

イベント同伴なしなら追加:

```text
入場時間
ブース
買いたい物
```

禁止:

```text
全入口を常時出さない
04.14の大きい説明カードを出さない
```

### 5.4 記録

目的: 今すぐ残す。

初画面:

```text
話す
撮る
動画
場所
ピン
メモ
散歩開始
```

保存先:

```text
小型表示のみ
例: 保存先: コタ通常散歩
```

禁止:

```text
保存先がないから記録不能にしない
Context説明を出さない
未確認箱を通常主役にしない
```

### 5.5 思い出

目的: 見返す / 整理 / 改善。

初画面:

```text
記録一覧
写真/動画
レビュー
次回改善
場所メモ
```

禁止:

```text
現地記録の主操作を混ぜない
```

## 6. ルート画面最終方針

ルートはキャンプ専用ではない。  
外出共通の移動エンジンとして扱う。

ただし、表示内容は属性で変える。

```text
キャンプ: チェックイン逆算 / 買い物 / SA / 余裕あり標準ギリギリ
ドライブ散歩: 到着時刻 / 駐車場 / 現地散歩開始 / 休憩
イベント: 入場時間 / 駐車場 / 会場まで / 物販時間
買い物: 店舗 / 開店時間 / 買うもの / 寄り道
下見: 候補地 / 駐車場 / トイレ / 犬OK / 写真メモ
```

04.15で実装するのは、まず04.8のキャンプルート復旧。  
他タイプのルートは、入口とデータ受け皿だけに留める。

### 6.1 ルート上部禁止

```text
同時利用カード
Context説明カード
表示中/記録中の長文説明
親子孫ツリー
外出属性一覧
未確認箱説明
```

### 6.2 ルート上部許可

```text
小型チップ1行だけ
例: 表示中: 赤城山キャンプ / 保存先: コタ散歩
```

## 7. シナリオ監査

04.15実装前に、次のシナリオを通す必要がある。

### 7.1 通常散歩

```text
予定なしでも記録できる
コタ通常散歩として保存できる
写真/GPS/ピン/音声/メモが保存できる
散歩後に思い出で見返せる
```

### 7.2 ドライブ散歩

```text
車で行く外出として扱える
ルートと駐車場を見られる
現地で散歩開始できる
GPSは現地散歩セッションへ入る
```

### 7.3 キャンプ

```text
04.8ルートが見やすい
買い物/料理/ギア/天気に入れる
場内散歩を子セッションとして開始できる
キャンプ中に次回キャンプ準備も見られる
```

### 7.4 イベント同伴あり

```text
ペット同伴として準備が出る
水/休憩/暑さ寒さ/カートが見える
会場内散歩/写真/メモを残せる
```

### 7.5 イベント同伴なし

```text
ペットを連れて行かない前提で見たいブース/買いたい物/写真メモが出る
次回ペット同伴できるかを記録できる
同伴あり用の水/カート/会場内散歩が主役にならない
```

### 7.6 予定なし記録

```text
今すぐ話す/撮る/場所/ピン/メモができる
保存先不明時だけ未確認箱
あとで外出プランへ紐付けできる
```
