# OUTBASE FIELD REBUILD 01
## 現地タブ・最初の1画面ラフ

これは完成版ではなく、現地タブの方向確認用ラフ。  
RC10〜RC13のように、モード一覧や仕様説明を画面に出さない。

---

## 1. 画面の考え方

現地タブは、読ませる画面ではない。  
現地で迷わず押す画面。

最初の1画面でやることは3つだけ。

```text
話す
撮る
場所を残す
```

コタ散歩、サイト調査、設営、撤収、ドライブは、最初からモードとして選ばせない。  
ただし、必要なショートカットとして下に自然に出す。

---

---

## 4. 裏側でやること

ユーザーには見せすぎない。  
でも、保存時には裏で分類する。

```text
うんち / おしっこ / 体調
→ 非公開コタ健康メモ

危険
→ 危険ポイント

サイト候補 / 木陰 / 水場
→ 場所カード候補

音声メモ
→ 内容・時刻・場所つき現地メモ

写真
→ 現地写真。あとで設営/サイト/料理/ギアに整理

現在地
→ 場所記録。必要ならGoogle Mapで確認
```

---

## 5. 状況によってショートカットを変える

最初の1画面に全部出さない。  
今の状況から、出す候補を変える。

### コタ散歩っぽい時

```text
[ うんち ] [ おしっこ ] [ 体調 ] [ 危険 ] [ 犬友達 ]
```

### サイト調査っぽい時

```text
[ サイト候補 ] [ 木陰 ] [ 水場 ] [ トイレ ] [ 炊事場 ] [ 危険 ]
```

### 設営っぽい時

```text
[ 配置写真 ] [ 足りないギア ] [ 困った ] [ 次回改善 ]
```

### 撤収っぽい時

```text
[ 乾燥 ] [ 忘れ物 ] [ 積込 ] [ 壊れた/汚れた ]
```

### ドライブっぽい時

```text
[ 出発 ] [ 休憩 ] [ 買い物 ] [ 渋滞 ] [ 到着 ]
```

---

## 6. 重要な判断

この画面の主役は「分類」ではない。  
主役は「残す」こと。

```text
現地では残すだけ。
分類は裏でやる。
整理はあとでやる。
```

---

## 7. 採用条件

この方向で作るなら、最低条件はこれ。

```text
・最初の1画面で迷わない
・話す / 撮る / 場所 が一番目立つ
・ショートカットは必要そうなものだけ
・非公開メモは混ざらない
・場所カード候補へ送れる
・ギア/改善/帰宅後/移動ログへ裏で送れる
・説明文を画面に貼りすぎない
```

以上。
