# OUTBASE BOOT v7.3 Core07.3 準備ワークスペース化

## 目的
Core07.2.1の「ボタンを押すと下に詳細が増える」構造を廃止し、準備トップを司令塔、料理/ルート/ギア/コタ/天気を専用作業画面として扱う。

## 採用判断
- 準備トップは短く維持する。
- 詳細はトップ下に追加しない。
- 各項目はフル画面ワークスペースで開く。
- ワークスペース内だけスクロールする。
- 戻るボタンで準備トップへ戻る。

## 修正内容
1. 天気・判断、料理、ルート、ギア、コタを全てワークスペース化。
2. 確認するラベルを「作業メニュー」に変更。
3. 天気判断コピー系導線は削除継続。
4. キャンプ予定は日程が近い順を維持。
5. 各ワークスペース内に入力/保存導線を配置。
6. 下ナビ・下余白がワークスペース操作を邪魔しないようCSSを調整。

## 画面構成
準備トップ：
- 準備中の予定
- 詳細天気・判断
- 次にやること3つ
- 作業メニュー（料理/ルート/ギア/コタ）

ワークスペース：
- ← 準備に戻る
- 項目名
- 判断/確認/入力/保存

## 不採用にした構造
- ボタン押下で下にラベルや詳細を追加する方式
- 折りたたみだけで情報を隠す方式
- 準備トップに全詳細カードを縦並びさせる方式
- 天気判断コピー/LINE共有導線
