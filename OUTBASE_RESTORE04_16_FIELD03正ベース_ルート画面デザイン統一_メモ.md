# OUTBASE RESTORE04.16 FIELD03正ベース ルート画面デザイン統一

## 目的
RESTORE04.15でルート画面は戻ったが、他タブの見た目がルート画面と揃っておらず、アプリ全体として格好悪く使いにくい状態だった。

04.16では、ルート画面を単独機能ではなく OUTBASE 全体のデザイン親として扱い、予定 / 探す / 準備 / 記録 / 思い出 の入口を同じ質感に統一する。

## 固定方針
- 04.8で良かったルート一画面性を維持する
- 04.13のContext Modelは内部だけに残す
- 04.15の外出属性モデルも内部だけに残す
- 大型説明カードは出さない
- ルート画面と違う白カードUIを増やさない
- 各タブホームは説明画面ではなく入口画面にする
- デザイン基準は `obHeroCompact` / `obStopMini` / `obDepartMiniBand` 系のルート画面質感

## 主な変更
- VERSION を `outbase-restore04-16-field03-route-design-unify-20260710` に更新
- service-worker キャッシュ名を04.16に更新
- manifest start_url を04.16に更新
- `ob416Hero` / `ob416Status` / `ob416ActionGrid` / `ob416MiniBar` / `ob416Panel` を追加
- top/nav を04.16クラスへ更新
- home / prep / field / calendar / discover / memory を04.16デザインで再上書き
- routePrepView は04.15で戻した04.8系の一画面ルートを維持

## 04.16で直接触らないもの
- ルート計算ロジック
- 出発逆算ロジック
- 現地セッション / GPS / 写真 / 動画 / 音声 / ピン / 欠測 / 画面ON維持
- Context Modelの内部保存構造
- 外出属性LOCKの考え方

## 確認観点
1. ルート画面が04.15同様に一画面で見やすいか
2. HOME / 予定 / 探す / 準備 / 記録 / 思い出 がルート画面の見た目に寄ったか
3. 白いカードだらけ、説明画面っぽさ、テスト画面感が減ったか
4. Context Modelや外出属性が画面上で出しゃばっていないか
5. 通常散歩 / ドライブ散歩 / キャンプ / イベント同伴ありなしの入口が残っているか

## 判定
ZIP候補。
ただし、最終合格は実機で見た目確認後。
