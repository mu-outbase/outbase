# OUTBASE 現地モード仕様書 v1

## 0. 前提

この仕様書は、RC11のように「ボタンを並べる」ためのものではない。  
OUTBASEの現地タブを、ユーザーが実際にキャンプ・散歩・移動中に迷わず使えるようにするための設計仕様である。

現地タブの目的は、次の1つに絞る。

> 今している行動に合わせて、あとで忘れる情報を最小操作で残す。

現地タブは、管理画面ではない。  
ギア一覧、タイマー一覧、GPS履歴、サイトMAP、記録一覧を最初から全部見せない。  
最初に出すのは「今なにしてる？」であり、そこから必要な操作だけを出す。

---

## 1. 現地タブ全体構造

### 1-1. 最初に表示するもの

現地タブを開いたら、最初にこれを表示する。

```text
今なにしてる？

・コタ散歩
・キャンプ場散歩
・設営
・撤収
・ドライブ
・自由記録
```

ここで選んだモードによって、画面の主役ボタンと保存先が変わる。

### 1-2. 共通で常に必要な表示

どのモードでも、画面上部には以下を表示する。

```text
・現在の主役プラン
・活動モード
・現在時刻
・今日の天気メモ
・記録中かどうか
```

主役プランは、どの画面でも切り替えられる。  
ただし、現地タブではプラン切替を目立たせすぎない。操作中の邪魔にしない。

### 1-3. 共通操作

どのモードにも共通で使える操作。

```text
・音声メモ
・カメラ
・現在地
・Google Map
・文字メモ
```

ただし、最初から全部同じ見た目で並べない。  
各モードで優先順位を変える。

## 2. コタ散歩モード

### 2-1. 目的

コタ散歩モードは、コタの通常散歩を記録するためのモード。  
目的は「散歩記録」ではなく、コタの安全・体調・次回の散歩判断に使う情報を残すこと。

### 2-2. 残す情報

```text
・散歩開始時刻
・散歩終了時刻
・歩いた距離
・歩いた時間
・現在地
・コタの体調
・うんち
・おしっこ
・暑そう / 寒そう / 疲れている
・危険ポイント
・犬友達
・良かった道
・避けたい場所
```

### 2-3. 最初に出す操作

```text
大きく出す：
・散歩開始 / 散歩終了
・音声メモ
・カメラ
・現在地

小さく出す：
・うんち
・おしっこ
・体調
・犬友達
・危険ポイント
・Google Map
```

### 2-4. 非公開にする情報

以下は自動的に非公開扱い。

```text
・うんち
・おしっこ
・体調
・病気っぽいメモ
・薬
・食欲
・疲れ
```

これらはレビュー、共有、場所カードには出さない。  
整理画面では「コタ健康メモ」として別枠にする。

### 2-5. 公開/共有に使える情報

```text
・良かった散歩道
・危険ポイント
・犬友達
・写真
・景色
・歩いた時間
・歩いた距離
```

### 2-6. 整理画面で使う情報

整理画面では、以下に変換する。

```text
・良かった散歩コース
・避けたい道
・危険ポイント
・コタが歩きやすい時間帯
・コタが疲れやすい条件
・再訪したい散歩場所
```

### 2-7. 次回提案に使う情報

```text
・暑い日は短めにする
・危険ポイントを避ける
・良かった道を候補に出す
・前回コタが疲れていた場合は距離を短くする
・犬友達がいた場所を候補にする
```

## 3. キャンプ場散歩モード

### 3-1. 目的

キャンプ場散歩モードは、キャンプ場内を歩きながら、次回再訪やレビューに使える場所情報を残すモード。

通常散歩ではない。  
主な目的は「キャンプ場の下見」である。

### 3-2. 残す情報

```text
・サイト候補
・景色が良い場所
・木陰
・水場
・トイレ
・炊事場
・ドッグラン
・危険ポイント
・ぬかるみ
・段差
・車通り
・コタが休める場所
・水場までの距離感
・サイトから施設までの動線
・また泊まりたい場所
・次は避けたい場所
```

### 3-3. 最初に出す操作

```text
大きく出す：
・音声メモ
・カメラ
・現在地
・Google Map

小さく出す：
・サイト候補
・景色
・木陰
・水場
・トイレ
・炊事場
・危険
```

### 3-4. 場所カードにする情報

以下は場所カード候補にする。

```text
・サイト候補
・景色
・木陰
・水場
・トイレ
・炊事場
・ドッグラン
・危険ポイント
```

場所カードには、以下を保存する。

```text
・場所名
・種類
・写真
・メモ
・現在地
・主役プラン
・訪問回数
・次回使いたいか
```

### 3-5. レビューに使う情報

```text
・水場が近い/遠い
・トイレが近い/遠い
・景色が良い
・木陰がある
・コタに向いている
・危険ポイントがある
・次回泊まりたいサイト
・次回避けたいサイト
```

### 3-6. 次回提案に使う情報

```text
・次はこのサイトが良い
・夏は木陰の近くが良い
・コタ連れならこの動線が良い
・水場が遠いので洗い物を減らす
・危険ポイントがあるので夜は注意
```

## 4. 設営モード

### 4-1. 目的

設営モードは、設営中の配置・時間・不足・困ったことを残すモード。  
目的は「設営を記録すること」ではなく、次回の設営を楽にすること。

### 4-2. 残す情報

```text
・設営開始時刻
・設営完了時刻
・設営にかかった時間
・配置写真
・テント/タープ配置
・車との距離
・リビング位置
・寝室位置
・コタの場所
・足りなかったギア
・使わなかったギア
・困ったこと
・風向き
・日差し
・地面状態
・次回改善
```

### 4-3. 最初に出す操作

```text
大きく出す：
・設営開始 / 設営完了
・配置写真
・音声メモ
・足りないギア

小さく出す：
・タイマー
・困ったこと
・次回改善
・現在地
```

### 4-4. ギア管理に戻す情報

以下はギア管理に戻す。

```text
・足りなかったギア
・使わなかったギア
・壊れたギア
・濡れたギア
・設営しにくかったギア
・次回積むべきギア
・次回不要なギア
```

### 4-5. 改善に戻す情報

```text
・この配置は良かった
・この配置は失敗
・風向きで向きを変えるべき
・コタの場所が悪かった
・キッチン動線が悪かった
・ペグ/ロープが足りなかった
・設営順を変えるべき
```

### 4-6. 次回提案に使う情報

```text
・前回この配置が良かった
・風がある日はこの向き
・このギアを忘れない
・このギアは使わなかった
・設営に何分かかったので出発時間を調整
```

## 5. 撤収モード

### 5-1. 目的

撤収モードは、帰る前後に発生する忘れ物・乾燥・積込・ゴミ・ケアを残すモード。  
目的は、帰宅後の整理と次回準備を楽にすること。

### 5-2. 残す情報

```text
・撤収開始時刻
・撤収完了時刻
・撤収にかかった時間
・乾燥が必要なもの
・濡れた幕
・濡れたロープ
・濡れたマット
・汚れたギア
・壊れたギア
・忘れ物確認
・ゴミ
・積込状態
・車載で困ったこと
・帰宅後にやること
```

### 5-3. 最初に出す操作

```text
大きく出す：
・撤収開始 / 撤収完了
・乾燥メモ
・忘れ物確認
・壊れた/汚れたギア

小さく出す：
・ゴミ
・積込
・タイマー
・音声メモ
・カメラ
```

### 5-4. 乾燥/修理/積込に戻す情報

```text
・乾燥が必要
・修理が必要
・買替が必要
・洗う必要あり
・車載位置を変える
・BOXに戻す
・積み忘れ注意
```

### 5-5. 次回忘れ物防止に使う情報

```text
・撤収時に忘れそうだった物
・最後に確認する物
・濡れていた物
・帰宅後にすぐ出す物
・車載で邪魔だった物
```

### 5-6. 次回提案に使う情報

```text
・撤収に時間がかかったので早めに始める
・乾燥が必要なので直営/乾燥サービスを優先
・壊れたギアを修理リストへ
・積込順を変更
・忘れ物チェックに追加
```

## 6. ドライブモード

### 6-1. 目的

ドライブモードは、キャンプ前後の移動中に発生する出発・休憩・買い物・渋滞・到着を残すモード。  
目的は、次回の出発時間、休憩場所、通り道買い物、帰り道判断に使うこと。

### 6-2. 残す情報

```text
・出発時刻
・到着時刻
・移動時間
・休憩場所
・給油
・買い物寄り道
・通り道コンビニ
・道の駅
・渋滞
・高速の混み具合
・帰り道の疲れ
・寄って良かった場所
・次は避けたい道
```

### 6-3. 最初に出す操作

```text
大きく出す：
・出発
・休憩
・買い物寄り道
・到着

小さく出す：
・給油
・渋滞
・道の駅/コンビニ
・Google Map
・音声メモ
```

### 6-4. 経路/休憩/買い物に使う情報

```text
・このコンビニが便利
・この道の駅が良い
・この時間は渋滞する
・給油タイミング
・買い物はここで済ませる
・休憩はここが良い
```

### 6-5. 次回出発時間に使う情報

```text
・前回出発時刻
・前回到着時刻
・渋滞した時間帯
・休憩回数
・買い物にかかった時間
・帰り道の混雑
```

### 6-6. 次回提案に使う情報

```text
・次回は30分早く出る
・このコンビニに寄る
・この道の駅は使える
・帰りはこの時間を避ける
・買い物を前日に済ませる
```

## 7. 自由記録モード

### 7-1. 目的

自由記録モードは、どのモードにも当てはまらないことを残すための保険。  
最初から使う主役ではない。

### 7-2. 残す情報

```text
・音声メモ
・カメラ
・現在地
・文字メモ
・動画
```

### 7-3. 後で整理する先

```text
・レビュー
・改善
・場所カード
・ギア管理
・買い物
・次回予定
```

## 9. 画面設計ルール

### 9-1. 現地タブで禁止すること

```text
・最初から全機能を見せる
・GPS履歴を上に出す
・タイマー一覧を上に出す
・サイトMAP管理を上に出す
・記録一覧を長く出す
・意味の分からない地図を出す
・ボタン名だけ並べる
```

### 9-2. 現地タブで必ず守ること

```text
・最初に「今なにしてる？」を出す
・モードごとに目的を表示する
・モードごとに押すボタンを変える
・非公開メモは混ぜない
・記録後にどこへ戻るかを決める
・詳細機能は閉じる
・現地では押すだけにする
```

## 10. 採用判定

この仕様を満たすまで、現地タブは完成扱いにしない。

特に重要なのは以下。

```text
・コタ散歩で、体調/排泄/危険/犬友達が扱える
・キャンプ場散歩で、サイト候補/水場/木陰/危険が扱える
・設営で、配置写真/不足ギア/困ったこと/改善が扱える
・撤収で、乾燥/忘れ物/積込/壊れたギアが扱える
・ドライブで、出発/休憩/買い物/渋滞/到着が扱える
・それぞれが整理/改善/準備/ギアに戻る
```

以上。
